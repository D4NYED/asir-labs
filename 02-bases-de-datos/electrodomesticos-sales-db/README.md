# Electrodomésticos Sales Database (ER + SQL)

## Objetivo
Diseñar el modelo de datos para una empresa de venta de electrodomésticos utilizando un Diagrama Entidad-Relación y su posterior implementación relacional en SQL.

## Entidades principales
- Productos
- Proveedores
- Clientes
- Empleados
- Ventas
- Detalles_Ventas (tabla intermedia N:M)

## Características del diseño
- Normalización de datos
- Claves primarias y foráneas
- Integridad referencial
- Resolución de relaciones N:M mediante tabla intermedia
- Escalable y modular

## Archivos
- er-diagram.png → diagrama entidad-relación
- schema.sql → estructura de la base de datos
- seed.sql → datos de ejemplo

## Tecnologías
SQL / MySQL / MariaDB
